# Engineer Navigator 開発ロードマップ

「SES × 生成AI ロードマップ」資料の *スキルの視覚化とAI教育・メンター制度* をアプリケーションとして実現する。

## プロダクトの中心思想

**週報がすべてのデータの源泉。**

社内に週報・業務報告のテンプレートは存在しないため、本アプリが「週報を書く場所」そのものになる。週報は人が読むための報告書ではなく、**AIが解析することを前提に設計された成長データの入力口**として再発明する（設計は [weekly-report.md](weekly-report.md)）。

```
        ┌─ スキル抽出 ──→ スキルマップ / 経歴書更新（単価交渉エビデンス）
週報 ───┼─ トーン解析 ──→ コンディションスコア → アラート
        └─ 詰まりごと ──→ AIメンターの学習提案 / ロールプレイ題材
```

エンジニアにとって「書くと得をする」（経歴書が勝手に育つ・相談できる）構造にすることで、報告義務ではなく自己投資として週報が回るようにする。

## フェーズ計画

### Phase 0: 基盤（〜2週間）
- [x] Next.js 16 + Prisma 7 + PostgreSQL(pgvector) プロジェクト構築
- [x] データモデル定義・マイグレーション（[data-model.md](data-model.md)）
- [x] AI層（`src/lib/ai/`）: Claude APIラッパー、プロンプト管理、コストログ
- [x] 認証: 開発用の簡易ログイン（cookie方式）→ 本番はGoogle Workspace SSO (sysnavi.co.jp 限定) ※未着手
- [x] シードデータ（スキルマスタ36件、デモユーザー3ロール）

### Phase 1: 週報 × スキルマップ自動更新（〜2ヶ月）— コアループ
プロダクトの心臓部。ここが回れば他機能はすべてこの上に乗る。

- [x] 週報フォーム（7問・必須3問、下書き保存つき）※自動保存は未実装
- [x] 提出時にAI解析パイプライン実行:
  - スキル・技術・工程の構造化抽出
  - 既存スキルマップとの差分検出 → `SkillSuggestion` 生成
  - ※実装済み・実LLMでの動作確認は ANTHROPIC_API_KEY 設定後
- [x] スキル承認UI: AI提案を本人がワンタップ承認/却下（承認でSkillHistoryに履歴）
- [ ] スキルマップ画面（カテゴリ別一覧は実装済み。レーダーチャート・時系列の成長グラフは未実装）
- [ ] 経歴書ビュー: スキルマップ + 承認済み実績から経歴書(スキルシート)を自動組版、営業向けエクスポート
- [ ] 「本番リリース経験」「AI活用実績」など単価交渉キーワードの自動ハイライト

**KPI**: 週報提出率 / スキル提案の承認率 / 経歴書が単価交渉に使われた件数

### Phase 2: コンディション検知アラート（〜1ヶ月）
Phase 1 の週報パイプラインに解析を1つ追加するだけで成立する。

- [ ] 自己申告スコア（天気メタファー）+ AIトーン解析スコアの2系統を毎週記録
- [ ] **変化検知**: 絶対値ではなく移動平均からの下降トレンド・急落を検知
- [ ] 「相談したい」フラグの即時通知
- [ ] 営業・管理者向けダッシュボード（担当エンジニアのコンディション一覧）
- [ ] アラート発火 → 対応記録（面談メモ）→ クローズのフロー

**設計上の必須事項（労務センシティブ）**:
- 解析していることを本人に明示（利用規約・初回オンボーディングで同意取得）
- コンディションデータの閲覧はADMINと担当営業のみ
- スコアを人事評価に使わないことをルールとして明文化

**KPI**: アラートからの早期面談実施数 / 突発離任の減少

### Phase 3: 資格・技術習得のAIメンター（〜1.5ヶ月）
- [ ] 学習コンテンツのRAG基盤（pgvector）: AWS/JSTQBシラバス、社内研修資料（booknavi研修資産の流用を検討）
- [ ] メンターチャットUI: 「理論 → 現場システムでどう適用するか」を必ず具体例つきで返すプロンプト設計
- [ ] 週報の「詰まりごと」からメンターが学習トピックを先回り提案
- [ ] 資格別学習プラン（試験日からの逆算カリキュラム、進捗トラッキング）

**KPI**: 資格合格数 / 週次アクティブ利用率

### Phase 4: 「一歩先の役割」シミュレーター（〜2ヶ月）
- [ ] リーダー職務定義書のシナリオ化（顧客からの無理な仕様変更、障害報告、メンバーフォロー等）
- [ ] AIが顧客役・メンバー役を演じるロールプレイチャット
- [ ] 終了後の自動フィードバック（職務定義書に照らした評価・改善点）
- [ ] 実施履歴がスキルマップに載る（「リーダーシップ演習◯回」が経歴書のエビデンスになる）

**KPI**: 現場リーダー登用数

## アーキテクチャ方針

- **単一Next.jsアプリ**で開始（フロント+API+Server Actions）。規模が出たらワーカー分離
- AI解析は提出時に非同期実行（失敗してもリトライ可能なジョブとして `AnalysisJob` で管理）
- LLM呼び出しはすべて `src/lib/ai/` 経由。プロンプトはコード管理し、モデル・トークン数・コストをログ
- 機密対策: 週報の自由記述から顧客名等を検出しマスキングする前処理を解析パイプラインに挟む（元資料「守りのAI武装」と整合）
- mykarteとは独立。将来必要になれば経歴書データをAPI連携で提供する側になる
